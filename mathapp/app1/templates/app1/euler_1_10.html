Problem 1: Multiples of Three and Five

Date posted: 28/09/2019

Problem rating: 5%

If we list all the natural numbers below 10 that are multiples of 3 or 5, we get 3, 5, 6 and 9. The sum of these multiples is 23. Find the sum of all the multiples of 3 or 5 below 1000.

When we first look at this problem, it is tempting to simply generate a list of all multiples of 3 below 1000, generate a list of all multiples of 5 below 1000, sum both lists and then add their result. But consider the first few multiples of 3 and 5:

  • 3: 3, 6, 9, 12, 15, 18, 21, ...
  • 5: 5, 10, 15, 20, 25 ...

Now we see that if we were to take the above approach, we would be double counting the numbers that are multiples of 3 and 5. To ensure we are not double counting, we can do the following:

  1. Initialize a sum variable to 0
  2. Run a loop for the integers 1 through 999
  3. If an integer is divisible by 3, add it to the sum
  4. Else if said integer is divisible by 5, add it to the sum
  5. Else, go on to the next integer

This way, we are only adding each integer to the sum at most once. The code for this is shown below.

total = 0

for number in range(1,1000):
    if number % 3 == 0:
        total += number
    elif number % 5 == 0:
        total += number

total = 233168, run time = 1.06 ms



Problem 2: Even Fibonacci Numbers

Date posted: 05/10/2019

Problem rating: 5%

Each new term in the Fibonacci sequence is generated by adding the previous two terms. By starting with one and two, the first ten terms will be: 1, 2, 3, 5, 8, 13, 21, 34, 55 and 89. By considering the terms in the Fibonacci sequence whose values do not exceed four million, find the sum of the even-valued terms.

We will define the Fibonacci numbers in a recursive manner, and then check if each number is even and less than four million. If it is, we can add it to the sum total. If it's not, then we either proceed to the next iteration of the sequence or terminate the loop. Keep in mind that we will have to remove the last number in the sequence from the sum since it will be larger than four million, which will cause us to exit our iterating loop. The general outline of our algorithm will look like this:

NOTE: In Python, the while loop will not add the final Fibonacci number to the sum, we don't have to subtract it at the end.

Here is the code:

F1 = 1
F2 = 2
total = 2

while F2 < 4000000:
    tempFib = F2
    F2 = F1 + tempFib
    F1 = tempFib
    
    if F2 % 2 == 0:
        total += F2

total = 4613732, run time = 1.03 ms



Problem 3: Largest prime factor

Date posted: 11/10/2019

Problem rating: 5%

The prime factors of 13195 are 5, 7, 13 and 29. What is the largest prime factor of the number 600851475143?

Recall that a prime number is a positive integer that has exactly two factors: one and itself. We can break this problem down into two steps:

  1. Generate a reasonably large list of prime numbers
  2. Test to see which of them divides 600851475143

The largest of these prime numbers will be our attempt at the solution. If we don't get the correct result, then we simply have to increase the size of our list of prime numbers. The code for this problem is shown below.

import math

primes = [2,3]

for i in range(5,100000,2):
    
    isPrime = True
    
    for j in range(3,int(math.sqrt(i)) + 1, 2):
        
        if i%j == 0:
            
            isPrime = False
            
            break
        
    if isPrime == True:
        
        primes.append(i)
        
divisors = []

for prime in primes:
    
    if 600851475143 % prime == 0:
        
        divisors.append(prime)
        
print(max(divisors))

Largest prime = 6857, run time = 348 ms



Problem 4: Largest palindrome product

Date posted: 17/10/2019

Problem rating: 5%

A palindromic number reads the same both ways. The largest palindrome made from the product of two 2-digit numbers is 9009 = 91 x 99. Find the largest palindrome made from the product of two 3-digit numbers.

Let's first think about how we could optimize our code. If we take a naive approach, we might try to multiply the number 100 by the numbers 100 through 999, check for and store any palindromes, then proceed to do the same thing for 101, etc. Immediately we see that the products of 100 x 101 and 101 x 100 are redundant, so we are wasting time. Instead, we only need to check the multiples that are greater than or equal to the number we are currently checking.

Once we have our product, how do we use our code to check if it is a palindrome? We can't do this directly with integers, but we could do it with string characters. For instance, suppose we wanted our program to check to see if the word "streetcar" was a palindrome. We could have our code compare the first letter in the string to the last letter, the second letter to the second last letter, etc. So how then do we make this possible for an integer like 10201? It's simple: we convert the type to string. If we determine that the string is a palindrome, we can store its integer type in a list that we will call at the very end of the code to return the maximum of all the palindromes we found. The code looks like this:

palindromes = []

for i in range(100,1000):
    
    for j in range(i,1000):
        
        test = True
        
        product = i*j
        pString = str(product)
        L = len(pString)
        
        for k in range(L):
            
            if pString[k] != pString[L-1-k]:
                
                test = False
                break
                
        if test == True:
            
            palindromes.append(product)

print(max(palindromes))

Largest palindrome = 906609, run time = 863 ms
